Project Summary

Quick scan of the core story.

Problem

Lead screening needed stronger differentiation to detect over-indebted profiles earlier.

Solution

A supervised classification model (LightGBM) trained on socio-demographic, financial-system behavior, and internal transactional signals.

Impact

Improved separation between good and bad leads to 11% and achieved ~83% overall accuracy.

Users

Risk and business teams using the score to support sustainable growth decisions.

Key Stakeholder Views

Artifacts and visuals used to communicate performance and impact.

Pipeline Development Flow

From cloud data to training, evaluation, and stakeholder-ready outputs.

  • Ingest & stage data in AWS S3 (multiple internal and external sources).
  • Feature building and dataset assembly via AWS Athena (SQL) over curated S3 tables.
  • Create a feature store-style dataset snapshot and export training-ready tables back to S3.
  • Spin up SageMaker training workers (compute instances) to run Python training pipelines.
  • Train LightGBM models, run parameter search iterations, and select the final candidate based on performance + stability.
  • Evaluate on Train/Test and Out-of-Time (OOT) periods; generate ROC/Gini and confusion-matrix metrics.
  • Compute SHAP explanations and variable importance summaries for model interpretability.
  • Package outputs (metrics, charts, and impact tests) into stakeholder materials and versioned artifacts.

Key Technical Highlights

Engineering choices that shaped the solution.

Cloud-native feature pipeline

Athena + S3 enabled scalable feature assembly and reproducible dataset snapshots.

Robust modeling approach

LightGBM chosen for strong performance on tabular data and resilience to common data issues.

Stability-first validation

Evaluation across development, test, and OOT windows to reduce performance surprises in production.

Explainability with SHAP

Global feature importance and driver analysis to support governance and stakeholder trust.

Lessons learned
  • Out-of-time evaluation is essential for risk models to validate stability beyond random splits.
  • Explainability accelerates stakeholder alignment and supports governance requirements.
  • Clear impact testing bridges the gap between model metrics and business adoption decisions.
  • Reproducible data snapshots (feature store-style exports) simplify iteration and auditability.
